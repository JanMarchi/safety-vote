# Story 2.1: Election Creation & Management

**Status:** Done
**Epic:** Epic 2 - Voting System & Election Management
**Story ID:** STORY-2.1
**Complexity:** 11 story points
**Priority:** High

---

## Summary

Implement election creation, scheduling, and management endpoints. Allow authorized users (RH/Admin) to create elections, set dates/times, configure rules (voting limits, abstention), and manage election status (draft → active → completed).

---

## Acceptance Criteria

### AC1: Admin/RH Can Create Elections
**Given:** User with 'rh' or 'admin' role
**When:** User calls POST /api/elections with valid data
**Then:** Election created, returns 201 with election object

### AC2: Election Has All Required Fields
**Given:** Newly created election
**When:** Election retrieved via GET /api/elections/:id
**Then:** Includes: id, title, description, company_id, status, start_date, end_date, max_votes_per_user, allow_abstention, is_secret, created_by, timestamps

### AC3: Election Status Lifecycle
**Given:** New election
**When:** Status transitions occur
**Then:** Follows: draft → active → completed (can also move to cancelled)

### AC4: Election Date Validation
**Given:** Election creation request
**When:** end_date ≤ start_date
**Then:** Returns 400 Bad Request with message "End date must be after start date"

### AC5: RH Can Edit Own Elections
**Given:** RH user created an election
**When:** RH calls PUT /api/elections/:id before election starts
**Then:** Election updated, returns 200

### AC6: Elections Isolated by Company
**Given:** User from Company A
**When:** User tries to access Company B's elections
**Then:** Returns 403 Forbidden (via RLS + API authorization)

---

## Scope

### In Scope
- CREATE election (POST /api/elections)
- READ elections (GET /api/elections, GET /api/elections/:id)
- UPDATE election (PUT /api/elections/:id) — before active
- Archive election (soft delete)
- Status transitions (draft/active/completed/cancelled)
- Configuration options (voting limits, abstention, secret ballot)
- Election scheduling (start_date, end_date validation)

### Out of Scope
- Real-time election updates (WebSocket)
- Email notifications for election start/end
- Bulk import elections from CSV
- Election templates
