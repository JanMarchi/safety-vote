## Summary

Implement secure voting system with AES-256-GCM encryption. Allow users to cast encrypted votes, prevent duplicate voting, and provide vote aggregation for results without exposing voter identity.

---

## Acceptance Criteria

### AC1: Eleitor Can Vote in Active Election
**Given:** Eleitor user, election status is 'active', within voting window
**When:** User calls POST /api/votes with election_id and candidate_id
**Then:** Vote cast successfully, returns 201, vote encrypted with AES-256-GCM

### AC2: Cannot Vote Outside Active Period
**Given:** Election not active or outside start/end dates
**When:** User attempts to vote
**Then:** Returns 403 Forbidden with message "Election is not currently active"

### AC3: One Vote Per User Per Election
**Given:** User already voted in election
**When:** User attempts to vote again
**Then:** Returns 409 Conflict with message "User has already voted in this election"

### AC4: Votes Encrypted with AES-256-GCM
**Given:** Vote cast successfully
**When:** Vote retrieved from database
**Then:** Contains encrypted_data (hex), iv (12 bytes), auth_tag (16 bytes), all valid

### AC5: Abstention Support
**Given:** Election allows abstention (allow_abstention=true)
**When:** User votes with candidate_id=null
**Then:** Abstention recorded, counted separately from candidate votes

### AC6: Results Aggregated (Anonymous)
**Given:** Election completed, results requested
**When:** Admin calls GET /api/votes/results
**Then:** Returns vote counts per candidate + abstentions, no voter identity exposed

## Implementation Notes

### Encryption Strategy

Uses AES-256-GCM (Galois/Counter Mode):
- **Algorithm**: AES with 256-bit key
- **Mode**: GCM (provides both encryption + authentication)
- **IV**: Random 96-bit (12 bytes) per vote
- **Auth Tag**: 128-bit (16 bytes) detects tampering

Process:
```
1. Generate random IV
2. Encrypt vote data (candidateId + timestamp)
3. Get authentication tag
4. Store: encrypted_data, iv, auth_tag
5. On decryption: verify auth tag (detects tampering)
```

### Vote Privacy Model

```
Database stores:
- voter_id (plaintext) → prevents duplicate voting
- candidate_id (plaintext) → for vote validation
- encrypted_data → what they voted for (private)

Admin perspective:
- Can see "User A voted"
- Cannot see "User A voted for Candidate X"
- Can only aggregate encrypted votes to tally
```

### Results Calculation

```
1. Retrieve all votes for election
2. For each vote: decrypt and extract candidateId
3. Tally: count votes per candidate
4. Separate: count abstentions (null candidate_id)
5. Return aggregated results (no voter linkage)
```

---

## Dev Notes

- Encryption key stored in VOTE_ENCRYPTION_KEY environment variable
- Key must be 32 bytes (256 bits) of random data
- Uses Node.js built-in `crypto.createCipheriv()` with 'aes-256-gcm'
- IV generation uses `crypto.randomBytes(12)` for GCM
- All encryption/decryption is synchronous (fast)
