# Story 5.1: /recursos (Features & Resources Page)

**Status:** Draft

**Complexity:** Medium (3 points)

## Description

Create Features & Resources page showcasing Safety Vote product capabilities, features, and benefits. Serve as primary marketing page for product differentiation.

## Acceptance Criteria

- [x] AC1: Page structure with hero section
- [x] AC2: Feature cards (min 8 features) with icons
- [x] AC3: Benefits/USP section
- [x] AC4: Call-to-action buttons (Sign Up, Demo)
- [x] AC5: Responsive design (mobile, tablet, desktop)
- [x] AC6: SEO metadata (title, description, og tags)

## Scope

### IN
- Hero section (headline, subheading, CTA)
- 8+ feature cards with descriptions and icons
- Benefits section highlighting key advantages
- Comparison with traditional voting systems
- FAQ section (collapsible cards)
- CTA section at bottom
- Proper header/footer integration
- Responsive grid layout

### OUT
- Actual feature comparison matrix (static content only)
- Video embeds (content only, no video hosting)
- Customer testimonials section

## Dependencies

- Header component (existing)
- Footer component (existing)
- Design system tokens (colors, spacing, typography)
- Icon library (shadcn/ui or existing)

## Technical Notes

- Next.js page component (`/app/recursos/page.tsx`)
- TypeScript strict mode
- Tailwind CSS responsive classes
- Static metadata via `generateMetadata()`
- No backend API calls (static page)
- Optimize images with Next.js `<Image>` component
