# Story 5.11: /status (System Status Page)

**Status:** Done

**Complexity:** High (4 points)

## Description

Create System Status page with real-time indicators of service health, incident reports, and maintenance schedule. Critical for platform reliability transparency.

## Acceptance Criteria

- [x] AC1: Real-time service status indicators (API, Auth, Database, Email)
- [x] AC2: Status color coding (Operational, Degraded, Down)
- [x] AC3: Incident history and resolved incidents log
- [x] AC4: Scheduled maintenance announcements
- [x] AC5: Uptime statistics and SLA information
- [x] AC6: Status subscription/notification CTA

## Scope

### IN
- Hero section with current system status
- Service status grid:
  - API endpoint status
  - Authentication service status
  - Database status
  - Email service status
  - Overall system status
  - Real-time status indicators (green/yellow/red)
- Status details per service:
  - Last checked timestamp
  - Response time
  - Uptime percentage
- Incident history:
  - Current incidents (if any)
  - Recent resolved incidents
  - Incident details: time, duration, impact
  - Root cause analysis (when available)
- Scheduled maintenance:
  - Upcoming maintenance windows
  - Expected duration
  - Affected services
  - Impact level
- Uptime statistics:
  - 30-day uptime percentage
  - 90-day uptime graph
  - SLA commitments
- Status page history (timeline of incidents)
- Subscribe to status notifications CTA:
  - Email alerts
  - SMS alerts (optional)
  - Slack integration
- Status page API/webhook (for status monitoring tools)
- Header/footer integration

### OUT
- SMS notifications
- Slack/Teams integration (separate)
- Historical data export
- Custom status page branding

## Dependencies

- Status monitoring service (Uptime Robot, Statuspage.io, custom)
- API for real-time status data
- Incident management system
- Email notification service
- Design system tokens
- Header/footer components

## Technical Notes

- Next.js page: `/app/status/page.tsx`
- API integration: `/api/status` endpoint
- Real-time updates: consider Server-Sent Events (SSE) or WebSocket
- Refresh status every 30-60 seconds
- Cache status data briefly (5-10 seconds)
- Responsive design for mobile viewing
- Performance: optimize for real-time updates
- Metadata for SEO
